# Деплой PWA приложения для осмотра пожарной безопасности

## Варианты деплоя

### 1. GitHub Pages (рекомендуется для открытых проектов)

1. Создайте репозиторий на GitHub
2. Загрузите все файлы проекта в репозиторий
3. В настройках репозитория (Settings > Pages) выберите "Deploy from a branch"
4. Выберите ветку (обычно main) и папку (usually / (root) или /docs)
5. Сохраните изменения

### 2. Netlify

1. Зарегистрируйтесь на [netlify.com](https://www.netlify.com/)
2. Нажмите "Add new site"
3. Выберите "Drag and drop your site folder" и перетащите все файлы проекта
4. Нажмите "Deploy Site"

### 3. Vercel

1. Зарегистрируйтесь на [vercel.com](https://vercel.com/)
2. Нажмите "Add New Project"
3. Выберите "Create a new project from Git repository"
4. Выберите ваш репозиторий и следуйте инструкциям

### 4. Собственный сервер

1. Загрузите все файлы в корневую папку веб-сервера
2. Убедитесь, что сервер настроен на обслуживание `sw.js` с правильными заголовками:
   ```
   Service-Worker-Allowed: /
   ```
3. Настройте MIME-типы файлов:
   - `.json` должен обслуживаться как `application/json`
   - `.html` должен обслуживаться как `text/html`
   - `.js` должен обслуживаться как `application/javascript`

## Настройка HTTPS

Для корректной работы PWA и Service Worker необходим HTTPS. Убедитесь, что:

1. Ваш домен настроен на использование HTTPS
2. SSL-сертификат действителен
3. Все ресурсы (CSS, JS, изображения) загружаются по HTTPS

## Проверка установки

После деплоя проверьте:

1. Доступность главной страницы по адресу: `https://ваш-домен/fire_safety_inspection.html`
2. Корректную работу Service Worker на вкладке DevTools Application
3. Возможность установки PWA через меню браузера
4. Работу всех функций приложения

## Настройка Google Apps Script

Не забудьте обновить URL в файле `fire_safety_inspection.html`:

- Замените `YOUR_SCRIPT_ID` на ID вашего развернутого скрипта
- Замените `YOUR_SHEET_ID` на ID вашей таблицы Google Sheets

## Проверка работоспособности

1. Откройте приложение в браузере
2. Проверьте, что отображается значок "Установить" в адресной строке
3. Проверьте, что все элементы интерфейса отображаются корректно
4. Проверьте, что можно создать и отправить тестовую запись
5. Убедитесь, что данные появляются в Google Sheets

## Troubleshooting

### Приложение не устанавливается

- Проверьте, что сайт использует HTTPS
- Убедитесь, что файл `manifest.json` правильно указан в HTML
- Проверьте, что `sw.js` зарегистрирован и активен

### Не синхронизируются данные

- Убедитесь, что вы заменили плейсхолдеры в API_URL и SHEET_URL
- Проверьте, что Google Apps Script развернут и доступен
- Убедитесь, что у скрипта есть доступ к таблице

### Не работают оффлайн-функции

- Проверьте, что Service Worker активен
- Убедитесь, что все необходимые файлы добавлены в кэш
