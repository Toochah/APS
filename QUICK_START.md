# Быстрый старт: PWA для осмотра пожарной безопасности

## Локальный запуск

### С помощью Node.js:
1. Убедитесь, что Node.js установлен (проверьте командой `node --version`)
2. Запустите сервер:
   ```bash
   node server.js
   ```
3. Откройте в браузере: http://localhost:3000/fire_safety_inspection.html

### С помощью Python (если Node.js недоступен):
1. Убедитесь, что Python установлен
2. В командной строке выполните:
   ```bash
   cd c:\workspace
   python -m http.server 8000
   ```
3. Откройте в браузере: http://localhost:8000/fire_safety_inspection.html

## Основные функции приложения

### 1. Форма осмотра
- Выберите дату и укажите ФИО инженера
- Выберите систему (пожаротушение, сигнализация, СПТ)
- Укажите тип дефекта из выпадающего списка
- Установите приоритет (низкий, средний, высокий, критический)
- Добавьте описание дефекта и рекомендации по устранению
- Сделайте фото или загрузите из галереи

### 2. Дашборд
- Просмотр статистики по количеству дефектов
- Диаграммы по системам, типам дефектов, приоритетам
- Таблица критических дефектов

### 3. Отслеживание ремонта
- Поиск и фильтрация дефектов
- Обновление статуса (новый, в работе, исправлено)
- Добавление комментариев к работам

## Тестирование функций

### Установка PWA
1. Откройте приложение в Chrome
2. Нажмите значок "Установить" в адресной строке
3. Подтвердите установку
4. Проверьте, что приложение появилось в списке установленных

### Работа офлайн
1. В DevTools включите режим "Offline" (Network tab)
2. Заполните и сохраните форму
3. Проверьте, что данные сохраняются в IndexedDB
4. Включите онлайн-режим и убедитесь, что данные синхронизируются

### Проверка Service Worker
1. Откройте DevTools → Application → Service Workers
2. Убедитесь, что сервис-воркер активен
3. Проверьте, что ресурсы кэшируются

## Настройка для работы с Google Sheets

### 1. Google Apps Script
1. Перейдите на https://script.google.com/
2. Создайте новый проект
3. Замените код на содержимое файла `google_apps_script.gs`
4. Замените `YOUR_SHEET_ID_HERE` на ID вашей Google таблицы
5. Опубликуйте как веб-приложение с доступом "Любой пользователь"

### 2. Настройка приложения
1. В файле `fire_safety_inspection.html` найдите строку:
   ```javascript
   const API_URL = 'https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec';
   ```
2. Замените `YOUR_SCRIPT_ID` на ID вашего опубликованного скрипта
3. Найдите строку:
   ```javascript
   const SHEET_URL = 'https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/gviz/tq?tqx=out:json';
   ```
4. Замените `YOUR_SHEET_ID` на ID вашей Google таблицы

### 3. Создание таблицы Google Sheets
1. Создайте новую таблицу
2. Добавьте заголовки в первую строку:
   - A1: ID
   - B1: Дата
   - C1: Время
   - D1: Инспектор
   - E1: Система
   - F1: Тип дефекта
   - G1: Приоритет
   - H1: Описание
   - I1: Фото
   - J1: Статус
   - K1: Рекомендации
   - L1: Дата исправления
   - M1: Примечания

## Устранение неполадок

### Приложение не загружается
- Проверьте, что используете HTTP/HTTPS (не открываете файл напрямую)
- Убедитесь, что все файлы находятся в нужных местах

### Service Worker не регистрируется
- Проверьте, что используете HTTPS (или localhost для тестирования)
- Убедитесь, что файл sw.js доступен по прямой ссылке

### Не синхронизируются данные
- Проверьте, что вы заменили ID скрипта и таблицы
- Убедитесь, что Google Apps Script опубликован и доступен

## Производственный запуск

Для размещения приложения в продакшене:

1. Загрузите файлы на хостинг (например, GitHub Pages)
2. Настройте Google Apps Script для серверной части
3. Убедитесь, что все пути к файлам корректны
4. Проверьте работу всех функций в боевых условиях
