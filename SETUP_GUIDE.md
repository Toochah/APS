# Настройка PWA для осмотра систем пожарной безопасности

## Обзор

Это PWA-приложение позволяет фиксировать результаты осмотров систем пожаротушения и пожарной сигнализации, а также отслеживать выполнение восстановительных работ через Google Sheets.

## Требования

- Современный браузер с поддержкой PWA (Chrome, Edge, Firefox)
- Аккаунт Google для работы с Google Sheets и Google Apps Script

## Настройка Google Apps Script

1. Перейдите на https://script.google.com/
2. Нажмите "Новый проект"
3. Замените весь код в файле `Code.gs` содержимым из файла `google_apps_script.gs`, созданного в этом проекте
4. Замените `YOUR_SHEET_ID_HERE` на ID вашей таблицы Google Sheets:
   - Откройте вашу таблицу Google Sheets
   - В адресной строке скопируйте ID из URL: `https://docs.google.com/spreadsheets/d/[ID_ТАБЛИЦЫ]/edit`
5. Нажмите "Сохранить" (иконка дискеты)
6. Нажмите "Развернуть" → "Новая развертка"
   - В выпадающем списке выберите "Веб-приложение"
   - В поле "Кто имеет доступ" выберите "Любой пользователь"
   - Нажмите "Развернуть"
7. Скопируйте предоставленный URL развернутого приложения

## Настройка Google Sheets

1. Создайте новую таблицу Google Sheets
2. Добавьте следующие заголовки в первую строку:
   - A1: ID
   - B1: Дата
   - C1: Время
   - D1: Инспектор
   - E1: Система
   - F1: Тип дефекта
   - G1: Приоритет
   - H1: Описание
   - I1: Фото
   - J1: Статус
   - K1: Рекомендации
   - L1: Дата исправления
   - M1: Примечания

## Настройка приложения

1. В файле `fire_safety_inspection.html` замените:
   - `YOUR_SCRIPT_ID` в переменной `API_URL` на ID вашего скрипта (часть URL после `/d/` и до `/exec`)
   - `YOUR_SHEET_ID` в переменной `SHEET_URL` на ID вашей таблицы Google Sheets

2. Загрузите файлы на веб-сервер или используйте GitHub Pages для публикации

## Использование приложения

1. Откройте `fire_safety_inspection.html` в браузере
2. Нажмите кнопку "Установить" в браузере для установки PWA
3. Используйте приложение для:
   - Проведения осмотров систем пожарной безопасности
   - Добавления фотографий и описаний дефектов
   - Отслеживания статуса ремонта дефектов
   - Просмотра статистики по дефектам

## Особенности приложения

- Автономная работа: данные сохраняются локально и синхронизируются при появлении интернета
- Адаптивный дизайн для использования на мобильных устройствах
- Визуализация данных в виде диаграмм
- Возможность фильтрации и поиска дефектов
- Отслеживание статуса выполнения работ

## Поля таблицы

| Поле | Описание |
|------|----------|
| ID | Уникальный идентификатор дефекта |
| Дата | Дата проведения осмотра |
| Время | Время проведения осмотра |
| Инспектор | ФИО сотрудника, проводившего осмотр |
| Система | Тип системы (пожаротушение, сигнализация и т.д.) |
| Тип дефекта | Категория обнаруженного дефекта |
| Приоритет | Приоритет устранения (низкий, средний, высокий, критический) |
| Описание | Подробное описание дефекта |
| Фото | Ссылка на фотографию дефекта в Google Drive |
| Статус | Текущий статус (новый, в работе, исправлено) |
| Рекомендации | Рекомендации по устранению дефекта |
| Дата исправления | Дата завершения ремонта |
| Примечания | Дополнительная информация |

## Безопасность

- Приложение работает только с теми таблицами, ID которых вы указали
- Все данные передаются по защищенному протоколу HTTPS
- Фотографии хранятся в Google Drive с ограниченным доступом
